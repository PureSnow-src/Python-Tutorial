跟随[幻灯片](./面向对象编程.pptx)学习，并完成下述习题。

## 习题 1：开发一个游戏角色类

某独立游戏项目正在开发原型版本，当前需要先完成一个最基础的“角色对象”模块。策划希望程序中可以创建不同角色，并让角色具备查看状态、受到伤害、回复生命、拾取道具等基本能力。

请你开发一个 `GameCharacter` 类，并基于该类创建若干对象，完成最简单的角色交互功能。

### 功能背景

在一个游戏系统中，角色通常拥有：

- 名称
- 生命值
- 攻击力
- 是否存活
- 随身携带的道具

虽然不同角色的数值不同，但从程序设计的角度看，它们都可以抽象为“游戏角色”这一类对象。

### 开发要求

请开发一个 `GameCharacter` 类，至少包含以下内容。

#### 1. 属性设计

在构造器 `__init__()` 中完成下列属性的初始化：

- `name`：角色名称
- `hp`：当前生命值
- `attack`：攻击力
- `is_alive`：是否存活（布尔值）
- `items`：角色当前拥有的道具列表

其中，`items` 建议初始化为空列表，例如：

``` Python
self.items = []
```

#### 2. 接口设计

请为该类至少提供以下公开方法：

- `show_status()`：输出当前角色的完整状态
- `take_damage(value)`：角色受到伤害
- `heal(value)`：角色回复生命值
- `pick_item(item_name)`：拾取一个道具并加入 `items`

要求说明：

- 当角色生命值降到 `0` 或以下时，应将 `hp` 置为 `0`，并把 `is_alive` 设为 `False`；
- 当角色已经死亡时，不再允许继续回复生命值；
- `pick_item(item_name)` 中可以直接把道具名作为字符串加入列表。

#### 3. 对象实例化与测试

请在主程序中至少创建两个角色对象，例如：

- 一个玩家角色 `hero`
- 一个怪物角色 `slime`

并调用其方法，演示以下操作：

- 创建对象；
- 玩家角色拾取两个道具；
- 怪物角色受到一次伤害；
- 玩家角色回复一次生命值；
- 输出两个角色对象的状态。

### 结果示例（格式不必完全一致）

``` Python
角色名称：Hero
生命值：120
攻击力：30
是否存活：True
当前道具：['木剑', '小药水']
```

### 开发提示

1. `take_damage(value)` 中可以先使用 `self.hp -= value`，再判断角色是否死亡；
2. `show_status()` 中可以直接使用 `print()` 输出对象属性；
3. 本题重点是体会：
   - 类是模板；
   - 对象是实例；
   - 方法是对象对外提供的行为接口；
   - 对象内部可以同时保存“数值状态”和“列表数据”。

### 可选提升

若你已经完成基础要求，可额外实现下列内容：

- 为类增加 `attack_target(other)` 方法，让一个角色对象攻击另一个角色对象；
- 为类增加 `__str__()` 方法，使得 `print(character_obj)` 时可以直接输出简洁信息；
- 为类增加 `__bool__()` 方法：当角色存活时返回 `True`，死亡时返回 `False`。

## 习题 2：开发一个背包系统类

某游戏项目准备开发一个简化版背包系统。当前阶段不考虑图形界面，也不接入数据库，只要求先用 Python 类完成背包对象的封装设计。

请你开发一个 `Inventory` 类，对外提供基础道具管理接口，并将部分内部实现细节隐藏起来。

### 功能背景

一个背包对象通常至少应包含：

- 背包名称
- 最大容量
- 当前道具列表

从业务角度看，外部系统应该可以调用“添加道具、移除道具、查看背包内容”等功能；但像“检查背包是否已满”“输出内部日志”等实现细节，更适合放在类的内部处理。

为了让你顺便复习字典，本题中每个道具建议用一个字典表示，例如：

``` Python
{"name": "小药水", "type": "consumable", "value": 20}
```

### 开发要求

请开发一个 `Inventory` 类，并满足以下要求。

#### 1. 属性设计

在 `__init__()` 中初始化下列属性：

- `bag_name`：背包名称
- `capacity`：最大容量
- `items`：当前道具列表

其中，`items` 建议使用列表保存多个道具字典。

#### 2. 公开接口

请至少提供以下公开方法：

- `add_item(item)`：向背包中添加一个道具
- `remove_item(item_name)`：按名称移除一个道具
- `show_items()`：输出当前背包中的全部道具
- `get_item_count()`：返回当前道具数量

要求说明：

- 当背包已满时，`add_item(item)` 不允许继续添加；
- `remove_item(item_name)` 只需删除第一个同名道具即可；
- `show_items()` 中建议使用 `for` 循环逐个输出道具信息；
- `get_item_count()` 应返回一个整数，而不是只打印结果。

#### 3. 私有接口

请至少设计一个私有方法，用于表示“仅供类内部调用”的实现细节。例如：

- `__is_full()`：检查背包是否已满
- `__find_item_index(item_name)`：查找某个道具在列表中的位置
- `__log(message)`：输出或记录内部日志

你可以任选其一，也可以实现多个。

#### 4. 特殊方法名

请至少额外实现下面两个中的一个：

- `__len__(self)`：使 `len(bag)` 可以返回背包中的道具数量；
- `__bool__(self)`：当背包非空时返回 `True`，空背包返回 `False`。

如果你愿意，也可以两个都实现。

#### 5. 对象测试

请在主程序中完成下列测试：

- 创建一个背包对象，例如 `player_bag = Inventory("PlayerBag", 3)`；
- 构造 3 个道具字典并依次加入背包；
- 尝试再加入第 4 个道具，观察程序输出；
- 移除一个指定名称的道具；
- 输出当前背包内容；
- 使用 `len(player_bag)` 或 `if player_bag:` 测试你实现的特殊方法。

### 结果示例（格式不必完全一致）

``` Python
添加成功：小药水
添加成功：木剑
添加成功：回城卷轴
添加失败：背包已满
当前背包内容：
1. 小药水 consumable 20
2. 木剑 weapon 10
```

### 开发提示

1. 删除道具时，可以先遍历 `self.items`，找到目标后再删除；
2. 若你实现 `__find_item_index(item_name)`，则 `remove_item()` 可以复用这个私有方法；
3. 本题重点是理解：
   - 哪些接口应当对外公开；
   - 哪些细节应当封装到类内部；
   - 特殊方法名可以让对象更自然地接入 Python 语法。

### 可选提升

若你已经完成基础要求，可继续补充：

- 实现 `use_item(item_name)` 方法：使用指定道具后将其从背包中删除；
- 实现 `__contains__(self, item_name)`，使得 `"木剑" in player_bag` 成立；
- 实现 `__str__()`，使对象打印时更易读。

## 提交说明

完成上述两题时，建议按以下顺序进行：

1. 先写类定义；
2. 再写构造器 `__init__()`；
3. 再补充公开方法；
4. 再补充私有方法或特殊方法；
5. 最后在主程序中创建对象并测试调用结果。

初学阶段应尽量做到：

- 类名、方法名见名知意；
- 一个方法只负责一件事；
- 先把最基本的功能写通，再考虑额外扩展；
- 测试代码不要省略，写完一个接口就立刻调用一次看看效果。
